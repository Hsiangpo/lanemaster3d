# LaneMaster3D 长期执行计划（终端版）

## 1. 执行原则

1. 第一优先：先把分数打出来，再进入论文主文写作。  
2. 全流程终端执行，不依赖UI交互。  
3. 每次实验只改一个关键变量，保证可归因。  

---

## 2. 环境与运行基线

### 2.1 远端连接

```bash
ssh <user>@<host>
tmux new -s lane3d
```

### 2.2 基础运行命令

```bash
# 训练
bash scripts/train_dist.sh configs/openlane/r50_960x720.py experiments/exp001 2

# 评测
bash scripts/eval_dist.sh configs/openlane/r50_960x720.py experiments/exp001/latest.pth experiments/eval001 2

# 恢复
bash scripts/resume.sh configs/openlane/r50_960x720.py experiments/exp001 2
```

### 2.3 监控命令

```bash
nvidia-smi
tail -f logs/train_*.log
```

## 3. 阶段计划

## 阶段A：基线打通（W1-W2）

### 目标

- 跑通公开基线训练与评测链路
- 固化实验命名、日志、checkpoint规范

### 任务

1. 校验数据路径与配置文件  
2. 完成100 step冒烟训练  
3. 完成一次完整评测并导出指标  

### 验收

- 可稳定生成日志与checkpoint
- 评测脚本输出完整指标字段

---

## 阶段B：创新点A（W3-W5）

### 目标

- 集成`GCA`并验证误差收益

### 任务

1. 将`GeometryConsistencyAdapter`接入锚点特征流  
2. 增加开关配置与对照实验  
3. 输出远距误差对比表  

### 验收

- 至少一组配置在`x_far/z_far`有明确改善

---

## 阶段C：创新点B（W6-W8）

### 目标

- 集成`GCS`并完成联合训练

### 任务

1. 接入几何循环一致性损失  
2. 调整损失权重并完成网格实验  
3. 输出稳定性对比（loss曲线、方差）  

### 验收

- 相比无GCS训练，收敛稳定性提升

---

## 阶段D：冲榜（W9-W10）

### 目标

- 达到并稳定超过`F1 62.9`

### 任务

1. 分辨率/骨干/迭代步联合搜索  
2. 固定最优配置做3种子复验  
3. 产出主结果表与切片分析  

### 验收

- 主指标超过阈值
- 复验结果波动可接受

---

## 阶段E：论文双轨素材（W11-W12）

### 目标

- 顶会轨与期刊轨共用素材沉淀完成

### 任务

1. 产出主表、消融表、失败案例图  
2. 整理复现实验细节与命令  
3. 形成双轨稿件大纲  

### 验收

- 可直接进入论文撰写阶段

---

## 4. 实验命名规范

命名格式：

`<日期>_<模型>_<创新开关>_<分辨率>_<seed>`

示例：

`20260217_lm3d_gca1_gcs1_960x720_seed3407`

- `f1`

- `x_close`
- `x_far`
- `z_close`
- `z_far`
- `fps`

---

## 6. 风险控制

1. 训练中断：使用`tmux`和`resume.sh`恢复  
2. 结果漂移：固定随机种子并锁定配置  
3. 创新无收益：保留可回退基线分支，及时止损
